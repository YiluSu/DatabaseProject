Installation

Installation is a four part process:

  1. Running the "admininformation" script
  2. Deploying to your catalina webapp folder
  3. Create the database tables and default admin user
  4. Logging in as the default admin

Running "admininformation"

The "admininformation" script is a short Java program. To run it execute the following commands (assuming you're in the root folder of the system):

  1. cd proj
  2. javac admininformation.java
  3. java admininformation (fill in the requested information)
  4. cd ..

Deploying

Before using the deploy script you'll have to set the CATALINA_BASE environment variable to point to your catalina folder. In most peoples cases this will be ~/catalina and the following command can be used:

export CATALINA_BASE=~/catalina

After setting the CATALINA_BASE environment variable run the deploy script using: sh deploy.sh. Now you'll be able to access the system at http://HOST:PORT_NUM/proj/ where HOST is the hostname of the server the system installed on and PORT_NUM is the port number that tomcat is running on.

Populating the database

Including in the system are two .sql files: setup.sql and init.sql. Running setup.sql will create all of the tables. Running init.sql will create all necessary indices as well as the default user. To run "setup.sql" and "init.sql", start "sqlplus" and run the following commands:

@setup.sql @init.sql

Logging in for the first time

After deploying the system you'll need to login as the default administrator. Once logged in you can change the default admin password and create additional admin and user accounts. The default admin credentials are "admin" (username) and "admin" (password).
